# Projeto: Sistema de Gestão de Tarefas

## Descrição
Este projeto é um sistema de gestão de tarefas desenvolvido para treinar habilidades técnicas em SQL Server. O objetivo é permitir o cadastro, acompanhamento e gerenciamento de tarefas, usuários e status, com foco em boas práticas de modelagem, consultas e automações com SQL.

## Estrutura do Projeto
- **Banco de Dados:** Scripts para criação das tabelas, procedures, views e triggers.
- **Exercícios:** Sugestões de desafios para cada etapa do projeto.

## Entidades Principais
- **Usuários:** Pessoas que podem criar e gerenciar tarefas.
- **Tarefas:** Atividades a serem realizadas, com status, prazos e responsáveis.
- **Status:** Situação atual da tarefa (pendente, em andamento, concluída, atrasada).

## Roteiro de Estudos
1. **Modelagem e Criação das Tabelas**
   - Crie as tabelas: Usuarios, Status, Tarefas.
   - Defina chaves primárias e estrangeiras.
2. **Inserção de Dados**
   - Insira usuários, status e tarefas de exemplo.
3. **Consultas Básicas**
   - Liste tarefas por usuário, por status, por prazo.
4. **Procedures**
   - Crie procedures para inserir, atualizar e excluir tarefas.
5. **Triggers**
   - Implemente triggers para atualizar status automaticamente.
6. **Views e Relatórios**
   - Crie views para tarefas atrasadas, tarefas por usuário, produtividade.
7. **Desafios Avançados**
   - Implemente controle de permissões, histórico de alterações, relatórios dinâmicos.

## Como Utilizar
1. Importe os scripts SQL no SQL Server Management Studio.
2. Execute os scripts na ordem sugerida.
3. Realize os exercícios e desafios propostos.

## Sugestão de Exercícios
- Listar todas as tarefas atrasadas.
- Contar tarefas por usuário e por status.
- Atualizar o status de uma tarefa via procedure.
- Criar uma view de produtividade por usuário.
- Implementar trigger para marcar tarefa como atrasada automaticamente.
